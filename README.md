# Репозиторий для pull request

* В своём аккаунте на *GitHub* создать копию репозитория **maxim-stoma/gitHomeWork** с помощью кнопки "Fork".
* Клонировать копию репозитория на локальный компьютер.
* **Создать новую ветку.**
* Добавить файл с инструкцией в новую ветку.
* Дополнить инструкцию разделами по работе с удалёнными репозиториями, pull request.
* Зафиксировать изменения (коммиты).
* Отправить изменения на GitHub.
* На сайте GitHub выполнить Pull request.

# **Шпаргалка по MARKDOWN, внесенная способом -  создания и слияния веток в Git.**

## **Заголовки.**

Заголовки нужны для обозначения и наименования глав, параграфов, пунктов, подпунктов, статей и т.п..

Чтоб выделить заголовок текста, нужно перед заголовком поставить знак решетка и пробел, можно выделить до шести подзаголовков, ставя соответствуещее количесвто решеток перед подзаголовками:

# H1
## H2
### H3
#### H4
##### H5
###### H6

Кроме того, заголовки можно обозначить подчеркиванием, но в это случае можно будет добавить только один подзаголовок:

H1
======
H2
------

## **Выделение.**

Курсив обозначается - *звездочками* или _подчеркиванием_ с обеих сторон текста:  
* *текст*  
* _текст_

Полужирный обозначается - двойными **звездочками** или __подчеркиванием__ с обеих сторон текста:  
* **текст**  
* __текст__

Комбинированное обозначается - **звездочками и _подчеркиванием_** с обеих сторон текста, или наоборт __подчеркиваниями и *звездочками*__ с обеих сторон текста:  
* **_текст_**  
* __*текст*__

Зачеркнутый текст обозначается -  двумя тильдами с обеих сторон текста:  
* ~~Уберите этот текст~~ 

## **Списки**

Чтобы создать списки нужно:

* ### Нумерованный список:
     1. перед началом списка с новой строки поставить цифру;
     2. после цифры поставить точку пробел и вводим текст. 

* ### Ненумерованный список:
     * перед началом списка с новой строки поставить звездочку;
     * после звездочки поставить пробел и вводим текст.

* ### Вложенные списки:  
     1. Внутри пунктов списка можно вставить абзацы с таким же отступом.  
     2. Чтобы вставить разрыв строки, но не начинать новый параграф, нужно добавить два пробела перед новой строкой. Этот текст начинается с новой строки, но находится в том же абзаце.

* ### Ненумерованный список можно размечать:  
     * звездочками  
     - минусами  
     + плюсами
    
Обозначение заголовков можно использовать совместно со списками, а также пункты вложенного ненумерованного списка будут изображаться другим фигурами так, как изображенно в примере выше.

## **Таблицы.**

Таблицы не являются частью Markdown, но многие обработчики, например Markdown Here и Github, поддерживают их. Они позволяют легко добавить таблицы в электронное письмо, в других случаях для этого нужно копировать их из другого приложения.

Вертикальные линии обозначают столбцы.

| Таблицы       | Это                | Круто |
| ------------- |:------------------:| -----:|
| столбец 3     | выровнен вправо    | $1600 |
| столбец 2     | выровнен по центру |   $12 |
| зебра-строки  | прикольные         |    $1 |

Внешние вертикальные линии (|) не обязательны и нужны только, чтобы сам код Markdown выглядел красиво. Тот же код можно записать так:

Markdown | не такой | красивый
--- | --- | ---
*Но выводится* | **так же** | _**клево**_
1 | 2 | 3

## **Цитаты**

> С помощью цитат очень удобно в письме обозначать исходный текст.
> Эта строка - часть той же цитаты.

Разрыв цитаты.

> Это очень длинная строка, но она будет правильно процитирована даже при размещении на нескольких строках. Продолжаем писать, чтобы эта строка не вмещалась на одной строке в любом окне. Кстати, в цитаты можно *вставлять* даже **Markdown**.

## **Горизонтальные линии**

Чтобы в строке появилась горизонтальная линия, нужно в этой строке друг за другом прописать символы от трёх и более штук:

---

Дефисы

***

Звездочки

___

Подчеркивания

## **Новая строка**

Для понимания работы разрыва строк, нужно проделать следующие действия:

Это начальная строка

Эта строка отделена от предыдущей двумя новыми строками и станет *отдельным абзацем*.

Это тоже отдельный абзац, но:
Эта строка отделена одной новой строкой, поэтому она находится в *той же строке*.

Это тоже отдельный абзац, но:  
Эта строка отделена двумя пробелами и одной новой строкой, поэтому она находится в том же абзаце, но в новой строке.

Примечание: 
*В Markdown Here разрывы строк ведут себя так же, как в GFM, поэтому не нужно вставлять по две новые строки.*
